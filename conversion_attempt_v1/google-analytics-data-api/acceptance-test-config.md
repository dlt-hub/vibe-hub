# Overview

This document provides a configuration guide for integrating with a third-party analytics API using dltHub's REST API source. The integration supports various data retrieval operations, including full and incremental data syncs, with a focus on handling different types of reports.

## Authentication

- **Type**: `api_key`
- **Secrets**: Use `dlt.secrets["api_key"]` to securely store and access your API key.
- **Location**: The API key is included in the query parameters of the request.

## Resources

- **Resource Names**: Cohort report, demographic interest report, demographic age report, demographic gender report, publisher ads ad unit report, publisher ads ad source report, publisher ads page path report, publisher ads ad format report.
- **Endpoint Path**: `/v1/resource`
- **HTTP Method**: GET (default)
- **Pagination Strategy**: Offset-based pagination with a limit of 100 records per request.
- **Data Selector**: The data is extracted from the "data" field in the API response.

## Error Handling

- **Retry Logic**: Implement retry logic for handling transient errors with exponential backoff.
- **HTTP Status Codes**: Handle common HTTP error codes such as 429 (Too Many Requests), 500 (Internal Server Error), 502 (Bad Gateway), 503 (Service Unavailable), and 504 (Gateway Timeout).
- **Max Retries**: Set a maximum number of retries to prevent infinite loops.
- **Backoff Factor**: Use a backoff factor to increase the wait time between retries.

## Schema

- **Fields**: Use neutral field names such as `analytics_id`, `customer_id`, etc.
- **Types**: Define field types and nullability based on the API response structure.
- **Nullability**: Ensure fields are marked as nullable if the API may return null values.
